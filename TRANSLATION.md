# API Documentation Translation Workflow

The English API documentation is generated by Sphinx from the docstrings. German translations are maintained separately in `.po` files — the source code remains in English.

## Directory Structure

```
docs/
  source/
    conf.py                      # Sphinx configuration
    api.rst                      # API reference structure
    locales/
      de/
        LC_MESSAGES/
          api.po                 # German translation of the API reference
          index.po               # German translation of the start page
  build/
    html/                        # English HTML documentation
    html/de/                     # German HTML documentation
```

## Accessing the Documentation

The built documentation is available as HTML in the `docs/build/html/` directory. The files can be opened directly in a browser:

- **English:** `docs/build/html/index.html`
- **German:** `docs/build/html/de/index.html`

Alternatively, a local web server can be started:

```bash
python -m http.server --directory docs/build/html 8080
```

The English documentation is then available at `http://localhost:8080` and the German version at `http://localhost:8080/de/`.

## Initial Setup

```bash
pip install -r requirements.txt
cd docs
make gettext
sphinx-intl update -p build/gettext -l de
```

## Building the Documentation

English:
```bash
cd docs && make html
```

German:
```bash
cd docs && sphinx-build -b html -D language=de source build/html/de
```

## Translation Workflow After Docstring Changes

When docstrings in the source code are changed, the translation needs to be updated:

1. Generate new `.pot` files:
   ```bash
   cd docs && make gettext
   ```

2. Update `.po` files:
   ```bash
   sphinx-intl update -p build/gettext -l de
   ```
   Changed entries are marked with `#, fuzzy`.

3. In `docs/source/locales/de/LC_MESSAGES/api.po`, find the marked entries, translate them, and remove the `#, fuzzy` flag.

4. Rebuild the German documentation:
   ```bash
   cd docs && sphinx-build -b html -D language=de source build/html/de
   ```

## The fuzzy Flag

Sphinx ignores entries marked with `#, fuzzy`. Only fully translated entries that are not marked as fuzzy appear in the German documentation. Untranslated entries (`msgstr ""`) are displayed in English.
